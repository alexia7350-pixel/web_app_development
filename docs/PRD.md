# 產品需求文件 (PRD) - 食譜收藏夾系統

## 1. 專案概述
- **背景與動機**: 現代人常在網路上看到喜歡的食譜，但缺乏一個集中且易於檢索的地方來保存。同時，使用者常常面臨「冰箱裡有這些食材，但不知道能做什麼菜」的困擾。因此，希望能有一個可以收藏食譜並根據現有食材推薦料理的平台。
- **目標用戶**: 
  - **一般人**：喜歡下廚、需要尋找靈感或整理自己食譜的使用者。
  - **管理員**：負責維護平台內容與社群秩序、管理使用者與分類。
- **核心價值主張**: 提供一個簡單操作的食譜收集庫，並透過「食材組合」功能幫使用者解決「不知道煮什麼」的日常痛點。

## 2. 功能需求
以下列出本系統的主要功能與對應的使用者故事：

### (1) 儲存食譜
- 作為 **一般人**，我希望 **能夠將網路上看到或自己創作的食譜儲存到個人收藏夾中**，以便 **日後下廚時可以快速找出來參考**。

### (2) 瀏覽與搜尋食譜
- 作為 **一般人**，我希望 **可以透過關鍵字、分類或標籤來搜尋所有公開發布的食譜**，以便 **找到心儀的料理做法**。

### (3) 從食材組合找食譜
- 作為 **一般人**，我希望 **能夠輸入手邊現有的幾樣食材組合找出相關食譜**，以便 **在不額外採買的情況下消耗掉冰箱剩餘的食材**。

### (4) 評價與留言互動
- 作為 **一般人**，我希望 **能夠對別人分享的公開食譜進行評分與留言**，以便 **提供實作心得或者改良建議給社群參考**。

### (5) 平台內容管理
- 作為 **管理員**，我希望 **能夠編輯、刪除違規食譜以及管理使用者帳號狀態**，以便 **維持平台內容的品質與社群的良好氛圍**。

## 3. 非功能需求
- **技術限制**: 本系統預計採用 **Flask + Jinja2 + SQLite** 作為基礎技術堆疊，以輕量化、快速開發為導向。
- **效能與安全考量**:
  - **效能**：食譜查詢（特別是食材組合查詢）應考量適當的資料庫索引或查詢最佳化，以確保搜尋回饋順暢。
  - **安全**：需防範常見的 Web 安全問題，如 SQL Injection 及 XSS；使用者的登入密碼必須經過安全的雜湊(Hash)演算法處理後再儲存在資料庫中。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (絕對要有)**
  - 使用者註冊、登入與權限區分（一般人 / 管理員）。
  - 基本的食譜 CRUD（建立、讀取、編輯、刪除）功能。
  - 將食譜加入我的收藏夾功能。
  - 從食材組合進行食譜搜尋的機制。
- **Should Have (應該要有)**
  - 首頁食譜牆與列表展現（包含縮圖、標題）。
  - 基本的分類與關鍵字搜尋功能。
  - 食譜圖片上傳與簡單儲存。
- **Nice to Have (加分項目)**
  - 針對食譜的給分評價與留言板。
  - 熱門或是本週推薦食譜排行榜。
